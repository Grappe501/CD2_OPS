# CD2_OPS — UX & A11y Standards (Module 21)

## North Star
If an 85-year-old grandparent can use it comfortably, everyone can.

## Mandatory behaviors
- All modals/drawers:
  - close with ESC
  - trap focus
  - visible close button
- All primary actions:
  - show loading state
  - show success confirmation
  - show friendly error on failure
- Forms:
  - field labels are plain language
  - examples are provided
  - no more than 6 fields per modal

## Visual
- Large type mode
- High contrast mode
- Avoid tiny icon-only buttons
- Keep the “Quick Add” in the same place everywhere

## Content
- Literal, non-technical language
- Always include “what happens next” after submit

## Tours
- Provide a guided tour chip on key dashboards
- Each tour: 2–6 steps, plain language

## Help Drawer
- Help drawer must be available on every page
- Include:
  - how to submit
  - how to use Discord
  - what happens next
